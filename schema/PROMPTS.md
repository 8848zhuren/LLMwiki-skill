# LLM Wiki Prompts — AI 提示词模板

> 本文件包含 AI 执行 Ingest 和 Query 时使用的提示词模板。使用 MiniMax M2.7 时，直接替换变量即可。

---

## 一、Ingest Prompt（新资料入库）

当用户说"帮我处理 [文件名]"时，AI 执行以下流程并使用本 Prompt：

```
## 任务
请处理以下原始资料，生成 LLM Wiki 页面。

## 原始资料路径
{filepath}

## 资料内容
{content}

## 执行步骤

### 1. 生成 Summary 页面
读取资料全文，生成一份结构化摘要，包含：
- 标题（从内容中提炼，或使用文件名）
- 一句话 summary（用于快速判断相关性）
- 核心要点（3-5 条关键信息）
- 详细信息（详细展开）
- 来源（原始文件路径）

保存到：`wiki/summaries/{slug}.md`

### 2. 提取 Entities
识别资料中提到的所有实体：
- 人物（人名、职位）
- 公司（公司名、品牌）
- 产品（产品名、功能）

对于每个实体：
- 检查是否已存在于 `wiki/entities/{type}/` 中
- 已存在 → 更新页面，补充新信息来源
- 不存在 → 创建新页面

### 3. 提取 Concepts
识别资料中的核心概念：
- 方法论（做事的方式、框架）
- 技术名词（专业术语、技术概念）
- 趋势（行业趋势、市场动向）

对于每个概念：
- 检查是否已存在于 `wiki/concepts/{domain}/` 中
- 已存在 → 更新，关联新来源
- 不存在 → 创建新页面

### 4. 建立交叉引用
在相关页面之间建立双向链接：
- Summary 页面链接到所有涉及的 Entity 和 Concept
- Entity 和 Concept 页面链接回 Summary
- Entity 之间、Concept 之间根据实际关联建立链接

### 5. 记录日志
在 `wiki/logs/` 目录下创建或更新日志文件，记录本次操作。

## 输出格式
完成后，请汇报：
- 创建了哪些页面
- 更新了哪些页面
- 建立了哪些链接
```

## 二、Query Prompt（知识查询）

当用户问知识性问题时，AI 使用本 Prompt：

```
## 任务
回答用户关于知识库的问题。

## 用户问题
{question}

## 回答要求

1. 先搜索相关 Wiki 页面（使用 grep 搜索关键词）
2. 读取所有相关页面的核心内容
3. 综合多页面信息，整理完整答案
4. 答案必须标注来源（列出参考了哪些 Wiki 页面及其路径）
5. 如果某些问题在 Wiki 中没有答案，用自己的知识回答，但明确说明"以下内容来自 AI 自身知识，不在 Wiki 中"

## 输出格式

### 回答
[综合多页面后的完整回答]

### 参考来源
- [[slug-1]] — wiki/summaries/xxx.md
- [[slug-2]] — wiki/concepts/tech/yyy.md
- （如有 AI 自身知识补充）AI 自身知识

### 相关页面
- [[slug-3]] — 用户可能也想了解
```

## 三、Health Check Prompt（定期检查）

每隔 1-2 周执行一次：

```
## 任务
检查 LLM Wiki 的健康度。

## 检查项目

### 1. 矛盾内容扫描
搜索 Wiki 中是否存在相互矛盾的信息（如同一公司不同页面的数据不一致）。

### 2. 过时页面标记
检查哪些页面长期未更新但可能已过时（超过 3 个月未更新的页面）。

### 3. 孤立页面检查
找出没有任何页面引用的页面（可能是死链接或废弃页面）。

### 4. 缺失链接补全
扫描所有 Summary 页面，确保都链接到了相关 Entity 和 Concept。

### 5. Inbox 清理
检查 inbox/ 目录下是否有未处理的碎片笔记，如有则处理。

## 输出格式

汇报：
- [ ] 矛盾内容：无 / 有（列出）
- [ ] 过时页面：数量 + 列表
- [ ] 孤立页面：数量 + 列表
- [ ] 缺失链接：数量 + 列表
- [ ] Inbox 处理：已清理 / 待处理

如有问题，提出修复建议，用户确认后执行更新。
```

## 四、MiniMax M2.7 使用建议

### Ingest 时
- 一次性读取完整资料（MiniMax M2.7 200K 上下文足够处理大多数文档）
- 不需要分段处理
- 中文资料直接输入，无需翻译

### Query 时
- 关键词搜索用 `grep -rl "关键词" wiki/`
- 读取相关页面时，一次性加载多个页面的核心内容
- 综合回答时保持逻辑连贯，避免幻觉

### Health Check 时
- 全文搜索矛盾关键词（如"增长率 30%"同时出现在多个页面）
- 检查同一 Entity 的多个页面间数据一致性
